Add a box-shadow to an element

The box-shadow property applies one or more shadows to an element.

The box-shadow property takes values for offset-x (how far to push the shadow horizontally from the element), offset-y (how far to push the shadow vertically from the element), blur-radius, spread-radius and a color value, in that order. The blur-radius and spread-radius values are optional.

Relative positioning

CSS treats each HTML element as its own box, which is usually referred to as the CSS Box Model. Block-level items automatically start on a new line (think headings, paragraphs, and divs) while inline items sit within surrounding content (like images or spans). The default layout of elements in this way is called the normal flow of a document, but CSS offers the position property to override it.

Setting an element to relative allows you to specify how CSS should move it relative to its current position. It pairs with the CSS offset properties of left/right and top/bottom. These say how many pixels, percentages, or ems to move the item away from where it's normal position.

For example this text has been moved 10px away from the bottom and 30px away from the left.

Changing an element's position to relative does not remove it from the normal flow - other elements around it still behave as if that item were in its default position.

Absolute positioning

Absolute positioning locks the element in place relative to its parent container. It removes the element from the normal flow of the document, so surrounding items ignore it

One changed with absolute positioning the element will be locked relative to its closest positioned ancestor. If you forget to add a position rule to the parent item the browser will keep looking up the chain and ultimately default to the body tag.

Fixed Positioning

This is a type of absolute positioning that locks an element relative to the browser window. Similar to absolute positioning, it's used with the CSS offset properties and also removes the element from the normal flow of the document. Other items no longer "realize" where it is positioned, which may require some layout adjustments elsewhere. The key differnce here is that the element WON'T move even when you scroll. This is why it is typically used with the navbar.

Float property: Another way to move an element left/right

Floating elements are removed from the normal flow of a document and pushed to either the left or right of their containing parent element.

It's commonly used with the width property to specify how much horizontal space the floated element requires. This works well if you want to set two elements next to eachother (multiple columns).

Overlapping elements - z-indexing

When elements are positioned to overlap, the element coming later in the HTML markup will, by default, appear on the top of the other elements.

To change this order, specify a "z-index" as an integer. The higher the number the higher tthe position is to the top.
